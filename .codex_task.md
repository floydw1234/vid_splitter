# Task: Wire BVF Binary Format into SmartBranching C# Plugin

## Context

The SmartBranching Jellyfin plugin currently reads `_branch.json` sidecar files.
The project has migrated to a binary `.bvf` format (Branched Video Format).
BVFReader.cs exists in `Assets/Scripts/` with full BVF parsing but is disconnected
from the plugin's playback pipeline.

## Scope — 4 Changes Only

### 1. Add ZstdNet NuGet Package

- Add `<PackageReference Include="ZstdNet" Version="1.4.*" />` to SmartBranching.Plugin.csproj
- This is needed by BVFReader.cs for zstd decompression of the manifest block

### 2. Replace ManifestScanner with BVFScanner

ManifestScanner.cs scans for `_branch.json` files. Replace it with BVFScanner.cs:

- Scan library for `.bvf` files (not `_branch.json`)
- Use `BVFReader.ReadHeader(filePath)` to get BVF metadata
- Use `BVFReader.GetSegments(filePath, profile)` to get resolved segments
- Keep the same `BranchManifest` model shape but populate from BVF data
- The `BranchManifest` model in Models/BranchManifest.cs should be updated to match
  the BVF manifest structure (profiles, segments with profile actions)

Key mapping:
- BVF manifest `profiles` → BranchManifest.Profiles
- BVF manifest `segments[].profiles.<profile>.action` → segment action resolution
- BVF segment index → segment timing/risk data

### 3. Wire BVFReader into SegmentServer

SegmentServer.cs currently has no-op implementations. Update it to:

- Load `.bvf` files instead of `_branch.json`
- Resolve segments using `BVFReader.GetSegments(filePath, profile)` for the user's profile
- Return `ResolvedSegment` objects with actual paths pointing to the `.bvf` file and
  segment data offsets (the path format should encode segment_id and data_offset)
- The segment server should be able to extract/serve segment data from the BVF file

For the segment path, use a format like:
`bvf://path/to/movie.bvf?seg_id=seg_001&offset=12345&length=67890`

This lets Jellyfin's streaming pipeline know where to seek in the BVF file.

### 4. Update Plugin.cs OnStartup

- Update the startup scan to look for `.bvf` files
- Log BVF-specific information (movie_id, segment_count, total_duration_ms)

## BVF Spec Reference

The BVF format is defined in `BVF_SPEC.md` at the repo root. Key points:
- File header: 64 bytes (magic, version, flags, offsets, segment count, total duration)
- Segment Index: array of 40-byte entries (segment_id, data_offset, data_length, duration_ms)
- Manifest: zstd-compressed JSON at manifest_offset
- Segment Data Blocks: 32-byte header + packet data (video/audio/subtitle packets)
- BVFReader.cs already implements the full parser — use it, don't reimplement
