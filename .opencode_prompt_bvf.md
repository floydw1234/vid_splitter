Implement the BVF muxer at vid_splitter/bvf_muxer.py.

Requirements from BVF_SPEC.md:
1. Write a BVF muxer module that accepts segment data and outputs a .bvf file
2. Follow the BVF spec exactly:
   - File header: 64 bytes - magic BVF with 0x01 byte + 4 reserved bytes, version (u16 major=1, u16 minor=0), flags (u32), index_offset (u64), index_length (u64), manifest_offset (u64), manifest_length (u64), segment_count (u32), total_duration_ms (u64), reserved (u32)
   - Segment index: flat array of 40-byte entries - segment_id (16 bytes null-padded), data_offset (u64), data_length (u64), duration_ms (u64)
   - Manifest: zstd-compressed JSON with bvf_version, movie_id, title, duration_ms, profiles, segments (id, start_ms, end_ms, tags, risk, profiles map with action/segment_id)
   - Segment data blocks: 32-byte header (magic SEG with 0x00, segment_id 16 bytes, codec_video u32, codec_audio u32, reserved u32) + variable packets

3. The muxer receives segment data from the analyzer in this format:
   - segments: list of {id, start_time, end_time, tags, risk, action}
   - duration_seconds from the analyzer
   - profiles dict from the analyzer

4. For the BVF muxer, implement:
   - BvfMuxer class with write_bvf method
   - Build the manifest JSON (zstd-compressed)
   - Build segment index entries (40 bytes each)
   - Write file header (64 bytes)
   - Support placeholder offsets that get backfilled
   - For segment data blocks: create stub blocks with SEG header and minimal marker packets
   - Include codec constants (H.264=0x00000001, H.265=0x00000002, AV1=0x00000003, AAC=0x00000100, etc.)
   - Packet types: video=0x01, audio=0x02, subtitle=0x03

5. Include comprehensive unit tests:
   - Test header construction (64 bytes, correct magic, version, flags)
   - Test segment index entry construction (40 bytes)
   - Test segment block header (32 bytes)
   - Test manifest JSON serialization and zstd compression
   - Test end-to-end: create segments, write bvf, read back and verify
   - Test that offset backfilling works correctly
   - Test classification mapping (safe/mature to risk field)
   - Test profile action mapping (play/swap/skip/mute)

6. Use struct for binary packing, zstandard for compression, json for manifest
7. Make the module importable with a main CLI for testing
