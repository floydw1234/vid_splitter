# Smart Branching — Jellyfin Plugin

A Jellyfin plugin that provides age- and profile-aware video playback from `.bvf` containers generated by the analyzer. Mature content segments are swapped, skipped, muted, or otherwise resolved based on the viewer's profile.

## Architecture

```
Python CLI (pre-processing)     ->    C# Jellyfin Plugin (playback)
----------------------------          --------------------------------
* analyze.py runs once per movie       * Runs inside Jellyfin server
* Generates Movie.bvf                  * Scans for .bvf beside movies
* Whisper + Safety Checker             * Serves resolved BVF segments
* BVF muxer embeds manifest/index      * No JSON sidecar required
```

## Prerequisites

### Python Side (Pre-Processing)
- Python 3.10+
- FFmpeg installed on the system
- GPU optional for faster Safety Checker inference

### C# Side (Jellyfin Plugin)
- Jellyfin Server 10.9.x
- .NET 9.0 SDK
- FFmpeg available to Jellyfin if playback paths invoke it

## Quick Start

### 1. Analyze a Movie

```bash
cd ~/Documents/codingProjects/vid_splitter
python -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python analyzer/analyze.py "/path/to/Movie.mp4" --model base
```

Output:
- `Movie.bvf`
- `frames/` extracted analyzer frames, safe to delete after analysis

### 2. Build the Plugin

```bash
cd csharp_plugin
dotnet build -c Release
```

Output: `bin/Release/net9.0/SmartBranching.Plugin.dll`

### 3. Install in Jellyfin

```bash
sudo cp bin/Release/net9.0/SmartBranching.Plugin.dll   /usr/share/jellyfin/data/plugins/
sudo systemctl restart jellyfin
```

### 4. Configure

1. Open Jellyfin Web UI.
2. Go to Dashboard -> Plugins -> Smart Branching.
3. Configure smart branching, default profile, thresholds, filler directory, and per-profile overrides.

### 5. Play

1. Place `Movie.bvf` alongside `Movie.mp4`.
2. Rescan the library.
3. Select the Smart Branch source when available.

## Directory Layout

/srv/jellyfin/media/

### Enable Plugin Logging

Add to Jellyfin's `logging.json`:

```json
{
  "Loggers": {
    "Jellyfin.Plugin.SmartBranching": "Debug"
  }
}
```

Logs appear in Jellyfin's log file, typically `/var/log/jellyfin/server.log`.

## BVF Manifest Reference

See `BVF_SPEC.md` and `PROJECT_SPEC.md` for the BVF manifest schema.

Key fields:
- `segments[].tags`: content tags such as `nudity`, `violence`, `language`, `fear`, `gore`
- `segments[].risk`: `safe`, `mature`, or `restricted`
- `segments[].profiles`: per-profile action and target segment mapping
- Profile actions: `play`, `swap`, `skip`, `blur`, `mute`

## Troubleshooting

### Plugin not appearing in Jellyfin
- Check that the DLL is in the plugins directory.
- Verify Jellyfin version matches `targetAbi` in `build.yaml`.
- Check Jellyfin logs for plugin load errors.

### No Smart Branch source appearing
- Ensure `Movie.bvf` is in the same directory as `Movie.mp4`.
- Verify the BVF parses with the Python reader or reference player.
- Check plugin logs: `grep SmartBranching /var/log/jellyfin/server.log`.

### Segments not playing
- Check file permissions; the Jellyfin user must be able to read the movie and `.bvf`.
- Verify the segment index offsets and lengths with BVF reader tooling.
- Ensure FFmpeg is installed and in PATH if the active playback path requires it.

### Analysis fails
- Check FFmpeg is installed: `ffmpeg -version`.
- For large files, use a smaller Whisper model: `--model tiny` or `--model base`.
- GPU helps significantly with Safety Checker; ensure CUDA is available when expected.
