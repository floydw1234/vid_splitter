# Smart Branching — Jellyfin Plugin

A Jellyfin plugin that provides age- and profile-aware video playback. Mature content segments are automatically swapped with fillers or skipped based on the viewer's profile.

## Architecture

```
Python CLI (pre-processing)     →    C# Jellyfin Plugin (playback)
─────────────────────────         ────────────────────────────────
• analyze.py runs once per movie  • Runs inside Jellyfin server
• Generates _branch.json          • Reads manifest on library scan
• Splits video into .ts segments  • Serves filtered stream natively
• Whisper + Safety Checker        • No external services needed
```

## Prerequisites

### Python Side (Pre-Processing)
- Python 3.10+
- FFmpeg installed on the system
- GPU optional (for faster Safety Checker inference)

### C# Side (Jellyfin Plugin)
- Jellyfin Server 10.9.x
- .NET 9.0 SDK
- FFmpeg (for segment streaming)

## Quick Start

### 1. Analyze a Movie (Python)

```bash
cd ~/Documents/codingProjects/vid_splitter

# Create virtual environment
python -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Analyze a movie
python analyzer/analyze.py "/path/to/Movie.mp4" --model base

# Output:
#   Movie_branch.json  (content manifest)
#   frames/            (extracted frames, can be deleted after)
```

### 2. Build the Plugin (C#)

```bash
cd csharp_plugin

# Build release
dotnet build -c Release

# Output: bin/Release/net9.0/SmartBranching.Plugin.dll
```

### 3. Install in Jellyfin

```bash
# Copy the DLL to Jellyfin's plugin directory
sudo cp bin/Release/net9.0/SmartBranching.Plugin.dll \
  /usr/share/jellyfin/data/plugins/

# Restart Jellyfin
sudo systemctl restart jellyfin
```

### 4. Configure

1. Open Jellyfin Web UI
2. Go to **Dashboard → Plugins → Smart Branching**
3. Configure:
   - Enable/disable smart branching
   - Default profile for unknown users
   - NSFW confidence threshold
   - Filler video directory
   - Per-profile filter overrides

### 5. Play

1. Place `_branch.json` files alongside your movies
2. Click "Play" on a movie with a manifest
3. A "Smart Branch" virtual source appears alongside the original
4. Select it to play the profile-filtered version

```
    ├── filler_safe_002.ts
    └── ...
```

## Creating Filler Videos

Filler videos should be valid MPEG-TS files. Create them with FFmpeg:

```bash
# 10-second black screen filler
ffmpeg -f lavfi -i color=c=black:s=1920x1080:d=10 \
  -c:v libx264 -pix_fmt yuv420p \
  -f mpegts filler_safe_001.ts

# 10-second "please wait" screen
ffmpeg -f lavfi -i color=c=blue:s=1920x1080:d=10 \
  -vf "drawtext=text='Please select a different movie':fontsize=48:fontcolor=white:x=(w-text_w)/2:y=(h-text_h)/2" \
  -c:v libx264 -pix_fmt yuv420p \
  -f mpegts filler_safe_002.ts
```

### Enable Plugin Logging

Add to Jellyfin's `logging.json`:

```json
{
  "Loggers": {
    "Jellyfin.Plugin.SmartBranching": "Debug"
  }
}
```

Logs appear in Jellyfin's log file (typically `/var/log/jellyfin/server.log`).

## JSON Schema Reference

See `PROJECT_SPEC.md` for the full `movie_branch.json` schema.

Key fields:
- `segments[].tags` — content tags (`nudity`, `violence`, `language`, `fear`, `gore`)
- `segments[].risk` — `"safe"` or `"mature"`
- `segments[].action` — `"play"`, `"swap"`, `"skip"`, `"blur"`, `"mute"`
- `segments[].swap_options` — per-profile swap targets

## Troubleshooting

### Plugin not appearing in Jellyfin
- Check that the DLL is in the plugins directory
- Verify Jellyfin version matches `targetAbi` in `build.yaml`
- Check Jellyfin logs for plugin load errors

### No virtual source appearing
- Ensure `_branch.json` is in the same directory as the movie
- Verify the JSON is valid and has the required fields
- Check plugin logs: `grep SmartBranching /var/log/jellyfin/server.log`

### Segments not playing
- Verify `.ts` files exist at the expected paths
- Check file permissions (Jellyfin user must be able to read them)
- Ensure FFmpeg is installed and in PATH

### Analysis fails
- Check FFmpeg is installed: `ffmpeg -version`
- For large files, use a smaller Whisper model: `--model tiny` or `--model base`
- GPU helps significantly with Safety Checker: ensure CUDA is available
